排序

快速排序

  • 核心思想：

    1. 设定分界值 a[l],a[r],a[(l+r)/2]a[l], a[r], a[(l + r) / 2]a[l],a[r],a[(l+r)/2]；
    2. 左边小于分界值，右边大于分界值
      • 双指针，两端开始，找到左边一个大于分界值，右边一个小于分界值，就将二者交换
      • 必须严格大于/小于，否则会 Memory Limit Exceeded；
    3. 递归

归并排序

  • 核心思想

    1. [L,R]⇒[L,mid],[mid+1,R][L,R]\Rightarrow [L,mid], [mid + 1, R][L,R]⇒[L,mid],[mid+1,R]
    2. 递归排序 [L,mid][L,mid][L,mid] 和 [mid+1,R][mid + 1,R][mid+1,R]
    3. 归并，将左右两个有序序列合并成一个有序序列

寻找一个数

  • 搜索区间：[0,a.size()−1][0,a.size()-1][0,a.size()−1]

  • 使用 l = mid + 1 和 r = mid -1 是因为 mid 已经被排除了

  • 模板

寻找左边界

  • 搜索区间：[0,a.size())[0,a.size())[0,a.size())

    • 所以 while 的判断条件是 l < r ，这样保证退出循环时，搜索区间是 [l,l)[l,l)[l,l) 空集

  • 为什么是 l = mid +1 和 r = mid，因为每次分割变成 [l,mid)[l,mid)[l,mid) 和 [mid+1,r)[mid+1,r)[mid+1,r)

  • 返回 r ，因为a[mid] == target时，有 r = mid。这也是为什么能够搜索左边界——当找到 target 后，并不是立刻返回，而是缩小上界

  • 如果 target 不存在，可以修改返回为 a[l] == target ? l : 1

  • 模板

寻找右边界

  • 搜索区间：[0,a.size())[0,a.size())[0,a.size())

    • 所以 while 的判断条件是 l < r ，这样保证退出循环时，搜索区间是 [l,l)[l,l)[l,l) 空集

  • 为什么是 l = mid +1 和 r = mid，因为每次分割变成 [l,mid)[l,mid)[l,mid) 和 [mid+1,r)[mid+1,r)[mid+1,r)

  • 返回 l - 1 ，因为a[mid] == target时，有 l = mid +1。这也是为什么能够搜索左边界——当找到 target 后，并不是立刻返回，而是缩小下界

  • 如果 target 不存在，可以修改返回为 a[l - 1] == target ? l - 1 : 1
